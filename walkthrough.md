# Active ETF Viewer 개발 워크스루

## 개요
이 문서는 Active ETF Viewer 애플리케이션의 개발 과정을 기록합니다. 초기 설정부터 데이터 파이프라인 구축, UI 구현 및 개선 사항을 포함합니다.

## 최근 변경 사항

### [2026-01-29] Config Refactoring & Web View
- **JSON Config 개선**: `activeetfinfos.json`의 운용사별 공통 인자(`--type` 등)를 Manager 레벨로 이동하여 ETF별 설정을 간소화했습니다.
- **Web View 기능**: 대시보드에서 ETF 제목을 더블 클릭하면 해당 상품의 웹 페이지(`view_url`)가 브라우저에서 열리도록 구현했습니다. 이 때 ID 값은 자동으로 치환됩니다.
- **Sidecar 통합**: 모든 ETF 스크래핑을 `get_pdfs` (Go) 사이드카로 통합하고, 레거시 바이너리(`get_time`, `get_koact`)를 제거했습니다.

### [2026-01-29] 관심 종목 (Favorites) 기능 추가
-   **DB 스키마 변경**: `etfs` 테이블에 `is_favorite` 컬럼을 추가하고, 데이터 시딩 시 해당값이 초기화되지 않도록 로직을 개선했습니다.
-   **Tauri 명령어**: 즐겨찾기 상태를 DB에 저장하고 불러오는 `toggle_etf_favorite`, `get_favorite_etfs` 커맨드를 구현했습니다.
-   **UI 통합**:
    -   **대시보드**: ETF 제목 옆의 별(★/☆) 아이콘을 통해 즐겨찾기를 토글할 수 있습니다.
    -   **사이드바**: 관심 종목으로 설정된 ETF는 목록에서 금색 텍스트와 아이콘으로 강조되어 쉽게 찾을 수 있습니다.
-   **상태 유지**: 즐겨찾기 상태는 로컬 DB에 저장되어 앱을 재시작해도 유지됩니다.

### [2026-01-29] 다중 날짜 일괄 업데이트
-   **Multi-Select DatePicker**: 업데이트할 날짜를 달력에서 여러 개 선택(Toggle)할 수 있도록 기능을 확장했습니다. 선택된 날짜는 빨간색으로 강조 표시됩니다.
-   **Batch Update**: 'Update' 버튼 클릭 시 선택된 날짜들에 대해 1초 간격으로 순차적으로 데이터를 수집합니다.
-   **Progress Indicator**: 버튼에 현재 진행 중인 작업 수(`Run (1/3)`)를 실시간으로 표시하고, 작업 중에는 버튼을 비활성화하여 중복 실행을 방지합니다.

### [2026-01-29] 로그 UI 개선
-   **로그 영역 토글**: 하단 로그 창을 접었다 펴는(Toggle) 기능을 추가하여, 필요 시 차트를 더 넓게 볼 수 있도록 개선했습니다.
-   **로그 관리**: 로그를 초기화할 수 있는 'Clear' 버튼을 추가하여 불필요한 로그를 정리할 수 있게 했습니다.

### [2026-01-28] 보유 종목 비교 기능 추가
-   **우측 사이드바 확장**: `Layout.tsx` 및 `Dashboard.tsx`를 수정하여 우측에 접을 수 있는 사이드바 패널을 추가했습니다.
-   **종목 리스트 (Holdings Table)**: 선택된 날짜의 전체 종목 구성(종목명, 코드, 비중)을 테이블로 표시하는 컴포넌트를 구현했습니다.
-   **비교 기능**: 테이블 하단에 비교 기능(DatePicker + 버튼)을 추가했습니다. 과거 특정 시점과 현재 시점의 포트폴리오를 비교하여, 신규 편입 종목(NEW)과 비중 변화(+, -)를 시각적으로 확인할 수 있습니다.
-   **UI 최적화**: 비교 영역의 공간 효율을 위해 날짜 선택기와 버튼을 한 줄에 배치했습니다.

## 주요 기능 (Features)

### 1. 대시보드 (Dashboard)
-   **ETF 차트**: 시계열 비중 데이터 시각화 (ECharts 사용).
-   **관심 종목 (New)**: 자주 보는 ETF를 별 아이콘으로 즐겨찾기 설정.
-   **데이터 업데이트**: Sidecar(Go)를 호출하여 실시간 웹 스크래핑 및 DB 업데이트(단일/일괄).

### 2. 사이드바 (Sidebar)
-   **운용사별 그룹화**: 아코디언 형태로 운용사 및 ETF 목록 관리.
-   **즐겨찾기 강조 (New)**: 관심 종목은 리스트에서 금색으로 표시되어 식별 용이.

### 3. 데이터 분석 (Analytics)
-   **기간 분석**: 선택된 두 날짜 사이의 비중 변화, 신규 편입/편출 종목 분석.
-   **종목 비교**: 단일 뷰에서 두 시점의 포트폴리오를 직관적으로 비교.
-   **로그 관리**: 작업 로그 확인 및 관리, 영역 최소화 기능.

<br>

## 기술 스택
-   **Frontend**: React, TypeScript, Vite, ECharts
-   **Backend**: Rust (Tauri), SQLite
-   **Data Collection**: Go (Sidecars)
### 차트 상호작용 및 레이아웃 개선 (2026-01-29)
*   **툴팁 개선**:
    *   화면 잘림 방지 (`appendToBody: true`, `confine: true`).
    *   **동적 정렬**: 해당 날짜의 비중 내림차순으로 종목 표시.
    *   **트리거 변경**: `trigger: 'item'`으로 변경하여 그래프 요소 위에서만 툴팁 표시 (빈 공간 제외).
    *   **가시성 확보**: `symbol: 'circle'` (opacity 0) + `emphasis` 효과로 마우스 호버 시에만 점이 나타나게 하여 미표시 문제 해결.
*   **레이아웃 버그 수정**:
    *   로그창 오픈 시 화면 밀림/헤더 잘림 해결 (`scrollTop` 사용, 차트 `ResizeObserver` 적용).
    *   차트 초기 로딩 시 크기 문제 해결.
    *   차트 최소 높이 550px로 복구.
*   **클릭 이벤트**:
    *   ZRender 클릭 이벤트를 통해 그리드 영역 전체에서 날짜 선택 가능하도록 개선.
